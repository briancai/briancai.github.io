Emacs的snippet

Veröffentlicht am 2019-08-13

简介

该模块用于提供快速输入代码片段功能。

配置

‘’’lisp
(require ‘yasnippet)

(define-key yas-minor-mode-map (kbd ““) nil)
(define-key yas-minor-mode-map (kbd “TAB”) nil)
(define-key yas-minor-mode-map (kbd “C-c k”) ‘yas-expand)

(setq yas-snippet-dirs
‘(“~/.emacs.d/MySnippets”))

(yas-global-mode t)
‘’’

snippet

  1. 创建snippet

‘’’lisp
C-c & C-n (M-x yas-new-snippet) ;; 创建新的snippet
C-c C-c ;; 保存snippet

  1. 例子

‘’’lisp
# contributor: jianxintest@test.com
# name: —
# –

${init}
‘’’

  1. 解释

“# – “ 以上的内容为注释，这些注释可能是指令或者元数据
指令格式类似 “# property: value”
若没有 “#–”，则所有内容均为snippet内容

  1. 指令

# key : 用来出发的指令
# name : snippet的描述
# condition : snippet展开的条件
# group : snippet菜单中的一组，通常是一个模式有多个snippet，可以组织到一个group中。
# expand-env : 为变量赋值的列表。在展开代码片段时，它可用于覆盖变量值
‘’’lisp

# note: example
# name: some line
# expand-env: ((yas-indent-line ‘fixed) (yas-wrap-around-region ‘nil))
# –
this is some line of txt in snippet
‘’’

# binding: 可以直接将展开的snippet指令绑定到emacs按键
# type: snippet或者command。如果type是command，那么snippet的内容会以lisp代码来执行。如果是snippet，那么代码片段的内容以模板语法来解释。

# uuid: 唯一标志
# contributor: 作者

  1. 模板语法
    这是最关键的内容。

纯文本
任意文本都可以作为模板的内容。除了美元符号\$和反引号`之外，其它都认为是纯文本。有需要时，使用\转义。
